emits

  • emits(emitter, event, ...__namedParameters): Promise<void>

  • Asserts that the emitter emits the specified event before the test ends. Returns a promise that resolves when the event is emitted. Note that waiting on the returned promise within a test can deadlock the test, if the event never emits, but the returned promise can be a handy way to pause a test until an event happens, if you are reasonably confident that it will fire.

error

  • error(er, ...__namedParameters): boolean

  • Assert that an error object is not provided.

    Like with doesNotThrow, you can also usually just throw the error, and tap will handle that reasonably.

    This is useful in cases where you just want to assert that a callback did not receive an error, without necessarily aborting the callback function entirely. Both the origin of the error and the location of the failed assertion will be printed in the test results.

    It is also used internally in resolves, doesNotThrow, and resolveMatch to show both the source of a raised error as well as the callsite where the assertion failed.
